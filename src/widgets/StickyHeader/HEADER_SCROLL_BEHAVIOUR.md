# Скрытие хедера при скролле: как это устроено

Документ описывает, как реализовано скрытие/показ хедера (логотипа) при скролле без дёргания страницы и с GrandMenu у верхнего края при скрытом хедере.

---

## Цели

1. **Скролл вниз** — хедер (логотип) скрывается; **скролл вверх** — показывается.
2. **Страница и скролл не дёргаются** при смене видимости.
3. **GrandMenu при скрытом хедере** визуально у верхнего края (без большого отступа).
4. **Плавная анимация** появления/скрытия.

---

## Почему не сработали предыдущие подходы

### Подход 1: схлопывание высоты (`height: 0`)

- У обёртки хедера при скрытии ставили `height: 0` вместо 48px.
- Высота документа менялась → контент под шапкой «прыгал» вверх → визуальный скачок.
- Компенсация через `window.scrollBy()` вызывала новые события `scroll` и повторные переключения видимости → страница снова дёргалась.

### Подход 2: только transform у хедера

- Обёртка хедера всегда 48px, хедер скрывался через `transform: translateY(-100%)`.
- Layout не менялся → скачков не было.
- Над GrandMenu оставался пустой блок 48px — меню выглядело «слишком далеко от верха».

---

## Идея финального решения

Использовать **только transform** и для хедера, и для GrandMenu: не менять поток документа (высота блоков постоянная), двигать только отрисовку.

- Хедер по-прежнему «уезжает» вверх через `translateY(-100%)` внутри фиксированной по высоте обёртки.
- **GrandMenu при скрытом хедере** сдвигаем вверх на те же 48px через `translateY(-48px)`.

В итоге: высота шапки в layout не меняется → длина страницы и скролл не меняются → нет дёргания; визуально при скрытом хедере GrandMenu оказывается у верхнего края.

---

## Реализация по частям

### 1. Хук `useHeaderScrollVisibility`

Файл: `src/hooks/useHeaderScrollVisibility.ts`

Логика только про «показать/скрыть» по скроллу:

- В зоне сверху (`scrollY <= topZone`) — хедер всегда видим.
- Ниже: по дельте скролла вниз/вверх и порогам (`thresholdHide` / `thresholdShow`) вызывается `setVisible(true)` или `setVisible(false)`.
- Обновляется `lastScrollY`, чтобы считать дельту от последней позиции.
- **Никакого** `scrollBy`, `scrollTo`, `flushSync` — скролл страницы из кода не трогаем.

Отсюда только булевый флаг «хедер видим»; вся визуализация — в стилях.

### 2. Стили: три слоя

Файл: `src/widgets/StickyHeader/StickyHeader.styled.tsx`

| Компонент          | Назначение                                                                                                                                                                                                                          |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **HeaderHideWrap** | Обёртка только под хедер (логотип). Всегда `height: 48px`, `overflow: hidden`. В потоке документа блок всегда занимает 48px → высота страницы не прыгает.                                                                           |
| **HeaderSlide**    | Внутри HeaderHideWrap, внутри неё сам Header. При скрытии: `transform: translateY(-100%)` — блок уезжает вверх и обрезается родителем. При показе: `translateY(0)`. `transition: transform 0.25s ease-out`.                         |
| **GrandMenuWrap**  | Обёртка вокруг GrandMenu. При скрытом хедере: `transform: translateY(-48px)` — меню сдвигается вверх и визуально оказывается у верхнего края. При видимом: `translateY(0)`. Та же `transition`, чтобы движение совпадало с хедером. |

Все три получают один и тот же флаг видимости хедера (например `$hidden` / `$headerHidden` из `!headerVisible`).

### 3. Разметка в StickyHeader

Файл: `src/widgets/StickyHeader/StickyHeader.tsx`

- Внутри `StickyHeaderWrap`: сначала `HeaderHideWrap` → внутри неё `HeaderSlide` с пропом `$hidden={!headerVisible}` → внутри `Header`.
- Потом, если нужно меню: `GrandMenuWrap` с `$headerHidden={!headerVisible}` и внутри `GrandMenu`.

Один источник правды — `headerVisible` из хука; по нему синхронно управляются и хедер, и сдвиг GrandMenu.

---

## Почему это не дёргается и не скачет

- **Layout не меняется:** ни у HeaderHideWrap, ни у GrandMenuWrap высота и позиция в потоке не зависят от видимости. Меняется только `transform`.
- **Скролл не трогаем:** никаких вызовов прокрутки в коде → нет лишних scroll-событий и повторных переключений.
- **Один и тот же флаг:** и хедер, и GrandMenu реагируют на одно и то же состояние, анимации совпадают по времени и направлению.

Итог: хедер плавно скрывается и появляется, GrandMenu при скрытом хедере визуально у верхнего края, страница и скролл остаются стабильными.
